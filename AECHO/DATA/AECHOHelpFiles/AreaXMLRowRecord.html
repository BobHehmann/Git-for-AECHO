AECHO V2 Help - XML Row/Record Display Area

<Return to Help's Home Page>

When a Line is selected from the ODF Display Area, AECHO locates the entire Row surrounding that Line, and copies its contents into this display area.

Rows may consist of one or more contiguous Lines. When an XML construct is bounded by '<o>...</o>' Tags, it is considered to be a Record, and its Row is defined as all contiguous Lines starting with the one containing the opening '<o>' Tag, and ending with the Line containing the closing '</o>' Tag. For all Lines that are not part of a Record, the Row is the same as the Line.

This area will present a vertical scroll-bar if the display of the Row requires more than two lines of text. Unlike the ODF Display Area, the XML Row/Record Display Area will word-wrap long lines of text, as it does not have room for a horizontal scroll-bar.

This area is read-only, but text may be freely selected and copied from it, suitable for pasting into any text editor.

The following image demonstrates the display of a Record-Row consisting of three Lines. As this exceeds the vertical space allowed, the area made its vertical scroll-bar visible, to allow access to any part of the text. The user has selected the Field '<f>12</f>' using a mouse-drag: this selection can now be Copied with a Ctrl-C, and later pasted into any program that accepts text.

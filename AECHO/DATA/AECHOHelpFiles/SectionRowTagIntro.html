AECHO V2 Help - Sections / Rows / Tags (aka Tables / Records / Fields)

<Return to Help's Home Page>

Hauptwerk ODFs are XML Documents, mostly organized to directly correspond with the common database concepts of Tables, Records, and Fields. This part of the Help Text uses these common database terms to demonstrate how an ODF is composed, and how AECHO interprets that composition.

Abstractly, an ODF is organized somewhat like this:
XML-Header            (Identifies this as XML)
ODF-Header            (A "Database" for an Instrument)
    Section-1         (A "Table" in that "Database"
        Row-1         (A "Record" in that "Table")
            Tag-a     (A "Field" in that "Record")
            Tag-b     (Another "Field" in the same "Record")
        Row-2         (Same structure as other "Records" in this "Table")
            Tag-a
            Tag-b
    Section-2         (A different "Table", with no "Records")
    Section-3         (Another "Table")
        Row-1         (Its first "Record")
            Tag-c     ("Fields" of the "Records" of this "Table")
            Tag-d
Each of these abstractions is implemented using standardized XML constructs recognized by AECHO. At all times, AECHO recognizes the type of construct the Cursor/Caret is located in (even if that Row has not been selected), and displays this on the bottom-left corner of its main form, in the Status-Bar. AECHO distinguished between the "_General" Section and the other 43 Sections, as "_General" has a somewhat different presentation at the Row/Tag (Record/Field) level than the other Sections: the remaining 43 Sections follow the abstract structure outlined above.

In XML, each of the Elements shown above is bounded by a Start-Tag and its corresponding End-Tag (see XML - a Brief intro... for more information) that define the extent of that Element's Content. The Content may include data values, Child Elements, or both. A somewhat more detailed abstraction might look like this:
<Section-1>
    <Row-1>
        <Tag-a>Tag-a's content...</Tag-a>
        <Tag-b>Tag-b's content........</Tag-b>
    </Row-1>
</Section-1>

In an ODF, the XML Header, ODF Header and Trailer, Section Headers and Trailers, and _General Section Data are all placed on dedicated single text-lines, ending with a line-break character. This is not a requirement of XML, but is presently a formatting convention of Hauptwerk. When selecting content by double-clicking in the ODF Display Area, AECHO treats this type of content as a single Row, comprised of a single Line.

Record content of Sections other than _General may span multiple lines i.e. the starting '<o>' and terminating '</o>' markup may be on different lines of text. AECHO defines this type of Row as all Lines containing the entire span of '<o>...</o>', however many that takes. Each individual Field within a Record is contained within a single Line i.e. a line-break never occurs between the Field's Start and End-Tags. However, line-breaks are allowed and often used between one Field's End-Tag and the next Field's Start-Tag, thereby resulting in a Row that spans multiple Lines. Lastly, within any single Record in a Compressed-ODF, only Fields containing non-default values are present: when HW reads a record, it internally supplies its defined default values for Fields that are officially part of a Section's Record definition, but not present in the ODF. AECHO presents the complete definition of a Record within the current Section, including the defaults HW will apply to absent Fields.

Hauptwerk convention is that all Rows start on their own Line, and end on their own Line - no Line contains any content of more than a single Row. This is only convention, and is not required by XML.

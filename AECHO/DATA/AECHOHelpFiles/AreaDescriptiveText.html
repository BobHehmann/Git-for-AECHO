AECHO V2 Help - Descriptive Text Area

<Return to Help's Home Page>

The Descriptive Text Area is predominantly used to display the descriptive text for the current Section. It is also used to display the Sections List (upon first loading an ODF, and subsequently on-demand), and occasionally used to display AECHO's Short-Form Help Text, if the Help HTML is unavailable.

The contents of this area can be sent to any system printer, physical or virtual, using menu > File > Print Descriptive Text... Contextually appropriate headers will be added to the top of the first printed page, and all formatting (font type, size, weight, color...) will be honored in the output.

The font-face and size of text displayed in this area can be standardized by clicking the Set Desc. Font button. This function preserves all other format attributes, such as bold, italic, underlined, and color. This button acts immediately upon the text currently displayed in the area, but has no ongoing effect if the content is reloaded. This command can be useful when pasting in content taken from other sources, that may use different font styles.

Text is this area is selectable, and can be freely cut and then pasted into other programs. It can also be freely edited, or content copied from elsewhere can be pasted into the area. Any such modifications will appear if the area is printed. While modifications to the Short-Form Help Text are transient, being overwritten the next time the text is displayed, changes to Section descriptive text can be permanently saved. The Save Desc. button will save Section-specific content back to the file system, allowing a user to customize their content. Note that such changes could be overwritten by AECHO's default content if the application is reinstalled or updated in the future.

The most common use of this area is to display the selected Section's descriptive text: this text lists all known Field Tags for the current Section (based on Hauptwerk V6.x specifications), showing their short-form abbreviated Tag-Names as seen in the Compressed-ODFs that AECHO works with, alongside their expanded Tag-Names as seen in a Full-ODF or in a database representation of an ODF. When defined, default values for the Fields are included - these are the values that HW fills in for an optional Field that is currently defined in the HW specification, but that does not appear in an ODF Record. Note that not all optional Fields have defined default values. Below the Fields that are presently in use, the descriptive text lists deprecated Tags, along with the HW version where they became obsolete. Obsolete Tags are simply ignored by Hauptwerk.

Upon initially loading an ODF, and subsequently on-demand using the command menu > Edit Mode > List Sections, the ODF is scanned from start to finish, and the area displays each Section's Number, Starting & Ending Line Number, Starting & Ending Character Position, and its Name.

If for some reason AECHO cannot open the extended HTML Help Text, it falls-back to displaying the Short-Form Help Text in this area.

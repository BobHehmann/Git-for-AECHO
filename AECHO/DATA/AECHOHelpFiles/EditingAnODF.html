AECHO V2 Help - Editing an ODF

<Return to Help's Home Page>
 
When first loaded, the ODF Display Area is read-only: its content can be displayed, searched, selected, and copied, but not altered. To make this area alterable, the user must first select the menu > Edit Mode > ODF Editing Enabled from the Menu-Bar. AECHO will issue an informational message that the text is now editable, change the background color of the Edit Mode menu from grey to red.

With editing enabled, the used may freely alter text just as with most common text editors: new text may be typed or pasted in; text may be deleted using the Backspace or Delete keys. Ctrl+X can be used to Cut (with a copy to the clipboard) selected text.

AECHO maintains an "unsaved modified text" indicator: it is visible on the Status-Bar as a colored box. The display is colored orange when there is no ODF; green when the current text is unmodified since it was loaded or last saved; red if there have been modifications made, but not yet saved. If the user requests an action that would dispose of unsaved modifications, such as exiting AECHO, closing the current ODF, or loading a new ODF, then the program will offer to save the users modified ODF.

When editable, the user may revert the ODF back to read-only by selecting menu > Edit Mode > ODF Editing Disabled.

If satisfied with the changes made, the user may select menu > File > Save As... to save the contents of the ODF Display Area back to a file. The system will default to the same directory from which the source ODF was loaded, but the user can select any accessible directory desired. Note that it is recommended to save to a new file name, so as not to overwrite a critical file. You may choose to overwrite an existing ODF file: AECHO will warn, but allow the operation if the user chooses to proceed.

Note that at this time, AECHO does not perform validation on the modified ODF, neither for XML syntax nor for adherence to Hauptwerk's specifications for an ODF.
